# AI 写作工作室 + 全局仪表盘实施文档

> 实施日期: 2026-04-22
> 状态: 已完成

## 概述

本次实施为 AI 短剧工作台添加了两个核心功能：

1. **全局仪表盘 + 左侧导航栏** - 统一的应用入口和功能导航
2. **AI 写作工作室** - 独立的剧本创作工具，支持 AI 辅助编辑

## 架构设计

### 数据模型

#### Script 表

```prisma
enum ScriptStatus {
  DRAFT     // 草稿，编辑中
  READY     // 定稿，可用于解析骨架和生成视频
  ARCHIVED  // 已归档
}

model Script {
  id          String       @id @default(cuid())
  userId      String
  user        User         @relation(fields: [userId], references: [id], onDelete: Cascade)

  // 未来可扩展：关联到 Project（从 Script 立项）
  projectId   String?
  // project   Project?     @relation(fields: [projectId], references: [id])

  title       String       @default("未命名剧本")
  content     String       @db.Text   // 剧本全文，纯文本
  status      ScriptStatus @default(DRAFT)

  // 版本预留（暂不启用版本表）
  version     Int          @default(1)

  createdAt   DateTime     @default(now())
  updatedAt   DateTime     @updatedAt

  @@index([userId])
  @@index([status])
}
```

**设计决策**：

- 暂时关联 `User` 表，后续可扩展关联 `Project`
- 预留 `projectId` 字段，支持从 Script 立项创建 Project
- `version` 字段为未来版本管理预留

## 后端实现

### API 路由 (`/api/scripts`)

| 方法   | 路径             | 说明                         | 权限            |
| ------ | ---------------- | ---------------------------- | --------------- |
| GET    | `/`              | 获取用户所有剧本             | 需登录          |
| GET    | `/latest`        | 获取最新草稿（无则自动创建） | 需登录          |
| POST   | `/`              | 创建新剧本                   | 需登录          |
| GET    | `/:id`           | 获取剧本详情                 | 需登录 + 所有权 |
| PUT    | `/:id`           | 更新剧本                     | 需登录 + 所有权 |
| DELETE | `/:id`           | 删除剧本                     | 需登录 + 所有权 |
| POST   | `/:id/ai-revise` | AI 修改代理                  | 需登录 + 所有权 |

### AI 修改代理逻辑

```typescript
POST /api/scripts/:id/ai-revise
Request: { content: string, instruction: string }
Response: { revisedContent: string }

流程:
1. 验证剧本归属
2. 调用方舟 API (deepseek-v3)
3. 记录 ModelApiCall（可观测性）
4. 返回修改后的内容
```

**System Prompt**:

```
你是一个专业的剧本编辑助手。根据用户的指令修改剧本内容，保持剧本格式和风格一致。
只返回修改后的完整剧本内容，不要添加任何解释或说明。
```

| 导出文件    | 下载 `.md` 文件                             |

**数据流**:

```
页面加载
    ↓
GET /api/scripts/latest
    ↓
用返回的 content 初始化编辑器
    ↓
用户编辑 → 防抖自动保存（2 秒）→ PUT /api/scripts/:id
    ↓
AI 修改 → POST /api/scripts/:id/ai-revise
    ↓
展示差异对比 Modal → 接受/拒绝
```

**预览渲染规则**:

```typescript
// Scene 标题 → 加粗高亮
if (/^Scene\s+\d+\./.test(line)) {
  return `<strong class="scene-title">${line}</strong>`
}

// 角色台词 → 角色名加粗
const match = line.match(/^([^：]+)："(.*)"$/)
if (match) {
  return `<strong>${match[1]}：</strong>"${match[2]}"`
}
```

## 后续扩展

### 1. 从 Script 立项

当用户在 AI 写作工作室完成剧本创作后，可以：

1. 点击"立项"按钮
2. 创建新 Project，并将 `projectId` 关联到当前 Script
3. 跳转到项目详情页继续后续流程

### 2. 版本管理

启用版本表 `ScriptVersion`：

```prisma
model ScriptVersion {
  id        String   @id @default(cuid())
  scriptId  String
  script    Script   @relation(fields: [scriptId], references: [id])
  version   Int
  content   String   @db.Text
  createdAt DateTime @default(now())
}
```

### 3. 实时协作

集成 WebSocket/SSE 实现多用户实时协作编辑。

### 4. Umo Editor 集成

将 `textarea` 替换为 Umo Editor，支持：

- 富文本编辑
- 更强大的预览
- 版本对比可视化
