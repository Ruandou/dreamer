# 后端第二阶段重构规划

## 第一阶段已达成（基线）

- [routes/](packages/backend/src/routes/) 下各路由以 Service 为主，**不再直接** `import` Prisma Client（仅个别类型如 [locations.ts](packages/backend/src/routes/locations.ts) 的 `Prisma` 类型）。
- 业务数据访问已按模块出现多份 `*Repository` + `*Service`；单例通过 `import { prisma } from '../lib/prisma.js'` 注入 Repository。

```mermaid
flowchart LR
  routes[Routes]
  services[Services]
  repos[Repositories]
  prisma[lib/prisma]
  routes --> services
  services --> repos
  repos --> prisma
```

## 第二阶段目标

在 [CODING_STANDARDS.md §2](docs/CODING_STANDARDS.md) 的「routes → services → repositories → Prisma」链路上，补齐 **尚未经过 Repository 的 Prisma 调用点**，并控制 **单文件体量**，便于单测与后续迭代。

---

## 优先级 A — 鉴权与归属（影响面大、改动集中）

**现状**：[plugins/auth.ts](packages/backend/src/plugins/auth.ts) 内大量 `prisma.*.findUnique` / `findFirst`，用于 `verifyProjectOwnership`、`verifyEpisodeOwnership` 等。

**建议**：

- 新增 **`OwnershipRepository`**（或按实体拆成极薄方法集，避免一个 500 行文件）：封装「按 id 查实体并带上 `project.userId` / `episode.project`」等查询，**不**写业务规则，只返回数据。
- 或复用已有 Repository：例如 `ProjectRepository.findFirstOwned`、`TakeRepository.findById` 等，在 auth 插件中只调 **Repository 的只读方法**。
- **plugins** 保持无 HTTP，仅依赖 Repository；单测可对 Repository mock，减少整库 Prisma mock。

**验收**：`auth.ts` 插件文件内无 `prisma.` 直接调用；现有 [auth-plugin.test.ts](packages/backend/tests/auth-plugin.test.ts) / [auth-plugin-jwt.test.ts](packages/backend/tests/auth-plugin-jwt.test.ts) 仍通过。

---

## 优先级 B — 队列 Worker（与 HTTP 解耦、Prisma 收口）

**现状**（均直接 `import { prisma } from '../lib/prisma.js'`）：

| 文件                                                      | 职责概要                          |
| --------------------------------------------------------- | --------------------------------- |
| [queues/video.ts](packages/backend/src/queues/video.ts)   | Take/Scene 状态更新、任务结果写回 |
| [queues/image.ts](packages/backend/src/queues/image.ts)   | 角色图/场地定场图创建与更新       |
| [queues/import.ts](packages/backend/src/queues/import.ts) | ImportTask、必要时创建 Project    |

**建议**：

- **Video**：在现有 [TakeRepository](packages/backend/src/repositories/take-repository.ts) / 新增 `SceneRepository` 小方法上扩展，或新增 **`VideoJobService`**（编排队列逻辑 + 调 Repository），`video.ts` 只 `add` + 调 Service。
- **Image**：与 [CharacterImageRepository](packages/backend/src/repositories/) / Location 写入对齐，抽 **`ImageQueueService`**（或扩展现有 image-generation 相关 service），队列文件只做 Bull 回调。
- **Import**：已存在 [ImportRepository](packages/backend/src/repositories/import-repository.ts)，将 [import.ts](packages/backend/src/queues/import.ts) 中剩余 `prisma.project.create` 等迁入 **ImportRepository** 或 **`ImportWorkerService`**。

**验收**：三个 queue 文件内无裸 `prisma.`（或仅剩不可避免的横切）；相关 worker 测试（如 [video-queue-worker.test.ts](packages/backend/tests/video-queue-worker.test.ts)、[import-queue-worker.test.ts](packages/backend/tests/import-queue-worker.test.ts)）仍通过。

---

## 优先级 C — Pipeline / 剧本任务大文件（分 PR 推进）

**现状**（行数大、仍含大量 Prisma）：

- [pipeline-executor.ts](packages/backend/src/services/pipeline-executor.ts)（约 457 行）
- [project-script-jobs.ts](packages/backend/src/services/project-script-jobs.ts)（约 428 行）

**建议**（顺序可调，但建议先 executor 再 script-jobs，因后者常依赖 pipeline 类型与任务状态）：

1. 将 **纯 Prisma 读写** 抽到 `PipelineRepository` / `PipelineJobRepository`（或扩展现有 [pipeline-repository.ts](packages/backend/src/repositories/pipeline-repository.ts)），与 [pipeline-orchestrator.ts](packages/backend/src/services/pipeline-orchestrator.ts) 的「步骤编排」边界划清。
2. `project-script-jobs` 中与 **Project / Episode / PipelineJob** 相关的重复查询，优先复用 **ProjectRepository** / Pipeline Repository，减少跨文件复制。

**验收**：行为与 API 不变；已有 [pipeline-routes.test.ts](packages/backend/tests/pipeline-routes.test.ts)、[project-script-jobs-async.test.ts](packages/backend/tests/project-script-jobs-async.test.ts) 等仍绿。

---

## 优先级 D（可选）— 规范对齐与长期债

- **`services/ai/`**：按 [CODING_STANDARDS.md §2](docs/CODING_STANDARDS.md) 将 [deepseek.ts](packages/backend/src/services/deepseek.ts)、与模型调用强相关的封装逐步迁入子目录（可单独 PR，避免与 A–C 混做）。
- **Repository 单例集中导出**：若多处 `new XRepository(prisma)` 重复，可增加 [repositories/index.ts](packages/backend/src/repositories/index.ts) 统一导出（仅组织性，非必须）。
- **handlers/**：仅在团队希望进一步薄化 `routes` 时再引入，第二阶段默认 **不强制**。

---

## 统一验收（每步）

1. 目标文件不再直接散落 `prisma.`（Worker/Plugin 同上）。
2. `pnpm --filter @dreamer/backend exec vitest run` 全绿；改动文件配套 related 测试或补 mock。
3. **ModelApiCall / ModelCallLogContext** 规则不变（[AGENTS.md](AGENTS.md)）。
4. 不批量变更对外 API JSON 形状。

---

## 建议实施顺序（汇总）

1. **A 鉴权 Repository 化**（横切、收益高）
2. **B 三队列**（video → image → import，可按风险从 import 最小开始）
3. **C pipeline-executor / project-script-jobs**（分多个小 PR）
4. **D** 按需排期
