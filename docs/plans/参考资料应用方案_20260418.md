## 二、应用位置与流程

### 1. 角色形象提示词生成

**应用文件**: `packages/backend/src/services/prompts/character-templates.ts`

**当前流程**:

```
用户点击"AI 生成分镜脚本"
  ↓
调用 applyScriptVisualEnrichment()
  ↓
AI 调用 visual-enrichment 模板 (批量提取角色+场地)
  ↓
为每个角色生成 images 数组:
  - base (基础定妆)
  - outfit (换装)
  - expression/pose (表情/姿态)
  ↓
调用 generateCharacterSlotImagePrompt() 为每个槽位生成提示词
  ↓
提示词存入 character_images.prompt 字段
  ↓
进入图片生成队列 (BullMQ)
  ↓
调用 Ark API (doubao-seedream-5-0) 生成图片
```

**应用方式**:

#### (1) 面部特征 → base 定妆提示词

**当前模板** (第 20 行):

```
【基础定妆】站立于纯色影棚背景（如中灰色）；七分身构图（膝盖以上）；
四段意合为一段：（1）主体与外貌（2）发型与标志（3）服装与姿态（4）构图与背景
```

**优化后** (加入面部特征分类):

```
【基础定妆 - 正面全身照】站立于纯色影棚背景（如中灰色）；正面全身构图（头顶到脚底完整可见）；
四段意合为一段、句间用句号分隔，整体不超过约150字：
（1）面部特征：脸型{如瓜子脸/方脸/圆脸}、眼睛{如杏仁眼/桃花眼/凤眼}、眉毛{如剑眉/柳叶眉}、
    鼻子{如高挺鼻/圆鼻头}、嘴唇{如薄唇/M字唇}、特殊标记{如痣/酒窝}
（2）整体外貌：{年龄}岁{性别}，气质{清冷/温润/凌厉}
（3）发型与标志：{发型}、发色{颜色}、{发饰/疤痕等标志}
（4）服装：{朝代/风格}{具体款式}{颜色}{材质}{纹样}
（5）构图与背景：正面全身，纯色中灰背景，{项目视觉风格}，电影级画质

参考词汇（供选择，勿堆砌）：
- 脸型：瓜子脸、方脸、圆脸、长脸、心形脸、鹅蛋脸、菱形脸
- 眼睛：杏仁眼、桃花眼、凤眼、圆眼、细长眼、狐狸眼
- 眉毛：剑眉、柳叶眉、平眉、拱眉、浓眉、细眉
- 鼻子：高挺鼻、圆鼻头、鹰钩鼻、朝天鼻、直鼻、蒜头鼻
- 嘴唇：薄唇、厚唇、M字唇、微笑唇、樱桃小嘴
- 发型：长发及腰、短发利落、双鬟、高髻、披发、束发

禁止只写剧情动作，须写「长什么样、穿什么」。
```

#### (2) 服装描写 → outfit 换装提示词

**当前模板** (第 50 行):

```
【换装】采用：保持该角色面部特征、发型与标志性细节完全不变，仅将服装更换为：……
```

**优化后** (加入服装分类体系):

```
【换装】采用：保持该角色面部特征、发型与所有标志性细节完全不变，仅将服装更换为：

服装描述公式：颜色 + 材质 + 纹样 + 款式 + 配饰

参考词汇：
- 款式：道袍、襦裙、劲装、鹤氅、留仙裙、直裰、褙子、马面裙
- 材质：灵蚕纱、天霞锦、云雾纱、丝绸、锦缎、粗布、麻衣
- 颜色：月白、朱砂、玄色、藕荷、黛色、紫金、青、白、黑
- 纹样：流云暗纹、八卦图、折枝花、龙凤纹、祥云纹
- 配饰：玉冠、步摇、丝绦、玉佩、储物袋、葫芦、长剑

示例：
- 清冷剑修：月白色灵蚕纱道袍，衣摆绣银色流云暗纹，交领右衽，宽袖长摆，腰间系青色丝绦悬挂长剑
- 温婉仙女：淡粉色留仙裙，外层轻纱如雾，内层丝绸贴身，裙摆层层叠叠如莲花绽放

整体不超过约120字，纯色影棚背景。
若已提供父级基础形象参考，须与之为同一人，勿写成新角色。
```

### 2. 场地定场图提示词生成

**应用文件**: `packages/backend/src/services/prompts/location-templates.ts`

**当前流程**:

```
AI 调用 visual-enrichment 模板
  ↓
为每个场地生成 imagePrompt (四段式)：
  1. 空间与环境
  2. 光影与氛围
  3. 构图与视角
  4. 风格与画质
  ↓
存入 locations.imagePrompt 字段
  ↓
调用 buildLocationEstablishingPrompt() 拼接
  ↓
进入图片生成队列
  ↓
调用 Ark API 生成定场图
```

**应用方式**:

#### (1) 场景分类 → 定场图提示词

**当前模板** (第 23-27 行):

```
每条 imagePrompt 须为中文，四段意合为一段连续文字、句间用句号分隔，整体不超过约120字：
1）空间与环境：核心建筑结构、标志性物体、材质与布局；
2）光影与氛围：依「时间」写光线方向、硬/柔与色温、情绪氛围；
3）构图与视角：推荐景别与角度（如广角全景、平视中景）；
4）风格与画质：融入下方「项目视觉风格」；并含画质词（如电影质感、8K超高清等）。
```

**优化后** (加入场景分类与示例):

```

【四段式结构】
1）空间与环境：核心建筑结构、标志性物体、材质与布局
```

## 三、实施步骤

### Phase 1: 更新提示词模板 (优先级: 高)

1. **修改 character-templates.ts**
   - 更新 CHARACTER_BASE_PROMPT_TEMPLATE: 加入面部特征分类、改为正面全身照
   - 更新 CHARACTER_OUTFIT_PROMPT_TEMPLATE: 加入服装描写公式与参考词汇

2. **修改 location-templates.ts**
   - 更新 VISUAL_ENRICHMENT_TEMPLATE: 加入场景分类速查表、材质/光线参考、示例
   - 更新 LOCATION_ESTABLISHING_TEMPLATE: 加入场景类型与描写模板

3. **修改 script-visual-enrichment.ts**
   - 更新角色提示词规则: 七分身→正面全身照，增加面部特征要求
   - 更新场地提示词规则: 增加场景分类引导

### Phase 2: 性能优化 (优先级: 高)

1. **拆分批量调用**
   - 当前: 1 次 API 调用处理所有角色+场地 (可能超时/超限)
   - 优化:
     - 第1次: 只提取角色名+场地名
     - 第2次: 为每个角色生成 images (并行，限制并发5)
     - 第3次: 为每个场地生成 imagePrompt (并行，限制并发5)

2. **限制输出长度**
   - 当前: maxTokens=4096
   - 优化: 限制每条 prompt 150 字，总数根据实体数量动态计算

### Phase 3: 测试验证 (优先级: 中)

1. **单元测试**
   - 测试 facial features 是否正确融入 prompt
   - 测试 outfit 公式是否正确应用
   - 测试 location 四段式结构

2. **集成测试**
   - 跑一次完整的 visual enrichment 流程
   - 检查生成的 prompt 是否包含具体面部特征
   - 检查 prompt 长度是否合理 (<150 字)

3. **实际测试**
   - 用测试项目跑 AI 生成分镜脚本
   - 检查生成的角色图片是否面部特征明显
   - 检查生成的定场图是否符合场景类型

## 五、注意事项

1. **提示词长度控制**
   - 角色 base: ≤150 字 (原 120 字，因增加面部特征需扩展)
   - 角色 outfit: ≤120 字 (原 100 字)
   - 场地 imagePrompt: ≤150 字 (原 120 字)

2. **AI 模型限制**
   - Ark API (doubao-seedream-5-0) 对中文支持良好
   - 但过长提示词可能忽略部分细节
   - 须控制质量词数量 (选2-3个即可)

3. **性能影响**
   - 更详细的提示词 = 更长的生成时间
   - 拆分批量调用会增加总 API 次数
   - 但单次调用更轻量，降低超时风险

4. **用户可编辑性**
   - 生成的 prompt 存入数据库后，用户可手动修改
   - 须在 UI 上显示 prompt 编辑框
   - 修改后重新入队生成图片
