### 2. Prompt Builder Pattern

**问题**: 重复的 `array.filter(Boolean).join()` 模式散落在多个文件

**解决方案**: 创建通用 PromptBuilder 类

```typescript
// Before
const parts = [title, location, time, desc].filter(Boolean)
return parts.join(', ')

// After
return new PromptBuilder(', ').add(title).add(location).add(time).add(desc).build()
```

**特性**:

- Fluent API 设计
- 自动处理 null/undefined
- 支持列表、条件、前缀
- 链式调用

**已重构文件**:

- [episode-service.ts](../packages/backend/src/services/episode-service.ts#L55-L68)

**文件**: [prompts/prompt-builder.ts](../packages/backend/src/services/prompts/prompt-builder.ts)

### 3. Strategy Pattern 应用

**问题**: pipeline.ts 中 40 行 switch 语句，违反 OCP 原则

**解决方案**: 策略映射表 + 注册机制

```typescript
// Before (40 lines)
switch (jobType) {
  case 'full-pipeline': { ... }
  case 'script-first': { ... }
  // ... more cases
}

// After (3 lines)
const handler = getPipelineJobHandler(jobType)
await handler(data)
```

**特性**:

- 开闭原则 - 添加新类型无需修改现有代码
- 运行时注册支持 (`registerPipelineJobHandler`)
- 更好的错误信息（列出可用类型）

**文件**:

- [queues/pipeline-job-strategies.ts](../packages/backend/src/queues/pipeline-job-strategies.ts)
- [queues/pipeline.ts](../packages/backend/src/queues/pipeline.ts) (从 144 行减少到 105 行)

### 4. 结构化错误日志

**问题**: catch 块缺少上下文信息，排障困难

**解决方案**: 创建统一的错误日志工具

```typescript
// Before
catch (error) {
  console.error('Script expansion failed:', error)
}

// After
catch (error) {
  logError('ScriptExpansion', error, {
    episodeId,
    operation: 'expand_episode_script'
  })
}
```

**功能**:

- `logError()` - 错误日志（含 stack trace）
- `logWarning()` - 警告日志
- `logOperationStart()` - 操作开始
- `logOperationComplete()` - 操作完成（含耗时）
