# 剧本模式检测失效修复计划\_20260418（修订版）

## 问题描述

**用户场景**：
用户上传了一个完整的36集剧本（`短剧剧本_天工开物_36集_改造方案.md`），但系统错误地走了**AI创作模式**（从零生成36集），而不是**解析模式**（faithful-parse解析现有剧本）。

**设计原则**：

- ✅ 前端只有**一个输入框**，用户输入创意或上传完整剧本
- ✅ **不做前端判断**，统一传入后端
- ✅ **后端自动检测**是创意还是剧本，然后分流

---

## 完整流程审查

### 当前流程（有问题）

```
用户输入/上传文件 → quickIdea
    ↓
创建项目 → { name, description: quickIdea }
    ↓
synopsis = null
    ↓
autogen=1 → 生成第一集
    ↓
runGenerateFirstEpisode()
  - 读取 project.description (完整剧本在这里!)
  - writeScriptFromIdea(description) ← ❌ 把完整剧本当创意
  - AI重新生成第一集
  - synopsis = AI生成的梗概 ← ❌ 覆盖
    ↓
runScriptBatchJob()
  - 读取 project.synopsis (AI梗概，不是原剧本!)
  - detectScriptMode(synopsis) ← ❌ 检测到错误内容
  - 判定为 ai-create
  - AI重新生成36集 ← ❌ 用户原剧本被忽略
```

### 正确流程（应该是）

```
用户输入/上传文件 → quickIdea
    ↓
创建项目 → { name, description: quickIdea }
    ↓
synopsis = null
    ↓
autogen=1 → 生成第一集
    ↓
runGenerateFirstEpisode()
  - 读取 project.description
  - detectScriptMode(description) ← ✅ 检测
  - 是完整剧本?
    ├─ YES → synopsis = description, 抛异常"应走解析"
    └─ NO → AI生成第一集, synopsis = script.summary
    ↓
runScriptBatchJob()
  - 读取 project.synopsis || project.description ← ✅ fallback
  - detectScriptMode(synopsis) ← ✅ 正确检测
  - 根据结果分流:
    ├─ faithful-parse → 解析现有剧本
    ├─ mixed → 混合模式
    └─ ai-create → AI生成
```

### 🔴 核心问题：runGenerateFirstEpisode 没有检测输入类型

**位置**：`packages/backend/src/services/project-script-jobs.ts:160-181`


4. 生成的梗概覆盖了原内容

```typescript
export async function runGenerateFirstEpisode(projectId: string) {
  const project = await projectRepository.findUniqueById(projectId)
  if (!project) throw new Error('PROJECT_NOT_FOUND')

  const idea = project.description?.trim() || project.name

  // ✅ 新增：检测 description 是否为完整剧本
  const detectionResult = detectScriptMode(project.description || '')

  if (detectionResult.mode === 'faithful-parse' || detectionResult.mode === 'mixed') {
    // description 里就是完整剧本，不需要AI生成
    // 直接存入 synopsis，让后续流程走解析
    console.log('[script-first] 检测到完整剧本，存入 synopsis 并引导解析')

    await projectRepository.update(projectId, {
      synopsis: project.description,
      storyContext: project.description.slice(0, 12000)
    })

    // 抛出特殊错误，告知前端应该走解析流程
    throw new Error('SHOULD_PARSE_SCRIPT_INSTEAD')
  }

  // 正常AI创作流程（创意输入）
  console.log('[script-first] 检测到创意输入，开始AI生成第一集')

  const { script } = await writeScriptFromIdea(idea, {
    modelLog: {
      userId: project.userId,
      projectId,
      op: 'generate_first_episode'
    }
  })

  const storyContext = [project.synopsis || script.summary, script.summary]
    .filter(Boolean)
    .join('\n')
    .slice(0, 12000)

  await projectRepository.update(projectId, {
    synopsis: script.summary,
    storyContext
  })

  const episode = await projectRepository.upsertEpisodeFirstFromScript(projectId, script)

  // 记忆提取...
}
```

### 修复4：优化评分标准

**文件**：`packages/backend/src/services/project-script-jobs.ts`

**修改位置**：第1088-1107行

```typescript
export function calculateOverallScore(script: string): number {
  let score = 0

  // 指标 1：分集标记 - 3分（最重要，提高到3分）
  const episodeMatches = script.match(/第\s*\d+\s*集/gi) || []
  if (episodeMatches.length >= 3)
    score += 3 // 至少3集
  else if (episodeMatches.length >= 1) score += 2

  // 指标 2：场景标记 - 2分
  if (/第?\d+[场景场]|scene\s*\d*/i.test(script)) score += 2

  // 指标 3：对话格式 - 2分
  if (/[""「"]|[\u4e00-\u9fa5]+[：:]\s*[""「"]/.test(script)) score += 2

  // 指标 4：字数 - 1分（提高阈值到5000）
  if (script.length > 5000) score += 1

  return score // 满分8分
}
```

**修改位置**：第1167行
