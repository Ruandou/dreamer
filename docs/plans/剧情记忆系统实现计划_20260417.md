# 剧情记忆系统实现计划

## 背景分析

当前项目的上下文管理存在以下问题：

1. **storyContext 字段过于简陋**：仅存储滚动剧情摘要（[project-script-jobs.ts:245](file:///Users/lei/Learn/dreamer/packages/backend/src/services/project-script-jobs.ts#L245)），无法区分角色、场地、事件、伏笔
2. **缺乏结构化记忆**：每次生成剧本/角色/场地时，LLM 无法查询历史记忆，容易重复或矛盾
3. **无法语义检索**：无法回答"哪些集出现了复仇相关的伏笔"这类问题
4. **上下文构建粗糙**：[script-writer.ts:74-78](file:///Users/lei/Learn/dreamer/packages/backend/src/services/script-writer.ts#L74-L78) 只是简单拼接字符串

## 系统架构

采用**双层记忆架构**：
- **结构化记忆**（PostgreSQL）：存储明确的剧情元素（角色、场地、事件、伏笔）
- **语义记忆**（embedding 向量）：存储剧情摘要的向量表示，支持语义相似度检索

### 数据模型设计

在 `schema.prisma` 中新增：

```prisma
// 记忆条目（核心表）
model MemoryItem {
  id          String   @id @default(cuid())
  projectId   String
  project     Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)
  
  // 分类
  type        MemoryType  // CHARACTER | LOCATION | EVENT | PLOT_POINT | FORESHADOWING | RELATIONSHIP
  category    String?     // 细分：major_character, minor_character, main_location, etc.
  
  // 内容
  title       String      // 简短标题："李明发现真相"
  content     String      @db.Text  // 详细描述
  metadata    Json?       // 结构化数据：{ episodeNum, characters, location, importance: 1-5 }
  
  // 向量检索（后续扩展）
  embedding   Float[]?    @db.RealArray  // 768维向量（OpenAI/DeepSeek embedding）
  
  // 关联
  relatedIds  String[]    @default([])   // 关联的其他 MemoryItem ID
  episodeId   String?     // 首次出现的集
  episode     Episode?    @relation(fields: [episodeId], references: [id])
  
  // 管理
  tags        String[]    @default([])   // ["复仇", "悬念", "关键转折"]
  importance  Int         @default(3)    // 1-5，5最重要
  isActive    Boolean     @default(true) // 是否仍影响后续剧情
  verified    Boolean     @default(false) // 是否经用户确认
  
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt
  
  @@index([projectId, type])
  @@index([projectId, isActive])
  @@index([projectId, importance])
  @@index([episodeId])
}

// 记忆快照（用于快速构建上下文）
model MemorySnapshot {
  id          String   @id @default(cuid())
  projectId   String
  project     Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)
  
  snapshotAt  DateTime  @default(now())  // 快照时间
  upToEpisode Int       // 包含到第几集
  
  // 压缩后的上下文
  summary     String     @db.Text  // 人类可读摘要
  contextJson Json               // 结构化上下文（供 LLM 使用）
  
  createdAt   DateTime  @default(now())
  
  @@unique([projectId, upToEpisode])
  @@index([projectId])
}

enum MemoryType {
  CHARACTER
  LOCATION
  EVENT
  PLOT_POINT
  FORESHADOWING
  RELATIONSHIP
  VISUAL_STYLE
}
```

### 阶段 3：上下文构建器

#### 3.1 Episode Writing Context Builder

**修改文件**：`packages/backend/src/services/memory/context-builder.ts`

替代现有的粗糙上下文构建逻辑：

```typescript
export async function buildEpisodeWritingContext(
  projectId: string,
  targetEpisodeNum: number
): Promise<string> {
  const memories = await memoryRepo.findByProject(projectId, {
    isActive: true,
    upToEpisode: targetEpisodeNum - 1
  })
  
  // 按重要性排序
  const sorted = memories.sort((a, b) => b.importance - a.importance)
  
  // 构建结构化上下文
  const context = formatContextForLLM({
    characters: sorted.filter(m => m.type === 'CHARACTER'),
    locations: sorted.filter(m => m.type === 'LOCATION'),
    events: sorted.filter(m => m.type === 'EVENT').slice(-10),  // 最近10个事件
    foreshadowings: sorted.filter(m => m.type === 'FORESHADOWING' && m.isActive),
    relationships: sorted.filter(m => m.type === 'RELATIONSHIP')
  })
  
  return context
}
```

async findSimilar(projectId: string, query: string, limit: number = 5) {
  const queryEmbedding = await this.generateEmbedding(query)
  return prisma.$queryRaw`
    SELECT * FROM "MemoryItem"
    WHERE "projectId" = ${projectId}
    ORDER BY embedding <=> ${queryEmbedding}::vector
    LIMIT ${limit}
  `
}

## 技术要点

1. **渐进式实现**：先做结构化记忆（阶段1-3），再做向量检索（阶段4）
2. **向后兼容**：保留 `Project.storyContext` 字段，新系统作为增强
3. **性能优化**：记忆快照机制避免每次都查询大量数据
4. **去重策略**：基于标题相似度 + 类型匹配进行记忆合并
5. **记忆衰减**：长期未提及的记忆自动降低重要性或标记为 inactive

## 预期收益

1. **剧本生成质量提升**：LLM 有完整的角色、场地、伏笔上下文
2. **避免重复**：新角色/场地提取时自动去重
3. **语义检索**：支持"找出所有复仇相关伏笔"等复杂查询
4. **可追溯性**：每个记忆关联到具体集数，便于回溯
5. **用户可控**：用户可编辑、确认、删除记忆
