# 核心功能补齐计划

> 日期：2026-05-01
> 目标：把重组后的"骨架"变成真正能用的短剧创作工具

---

## 一、模板库 → 大纲生成（P0）

### 问题

当前 `/api/templates/:id/generate-outline` 的 Prompt 直接输出 JSON，LLM 可能：

1. 输出格式不严格（包裹在 markdown 代码块里）
2. 集数不够或超量
3. 字段缺失（缺少 hook/cliffhanger）
4. JSON 解析失败

### 任务

- [ ] **1.1 强化 Prompt 模板**
  - 添加 Few-shot 示例（1-2集完整示例）
  - 明确输出格式约束（纯 JSON，无 markdown）
  - 添加 "自检指令"：要求 LLM 检查集数、字段完整性

- [ ] **1.2 添加 JSON 修复层**
  - 提取 markdown 代码块中的 JSON
  - 使用 `json-repair` 或正则修复常见错误（尾随逗号、缺失引号）
  - 验证结构：每个 episode 必须有 episodeNum/title/synopsis/hook/cliffhanger

- [ ] **1.3 集数对齐**
  - 如果生成集数不足，用模板最后一幕的节拍循环填充
  - 如果超量，截断并确保最后一集有收尾感

- [ ] **1.4 前端加载状态**
  - 生成时显示进度（"正在构思第 X 集..."）
  - 失败时允许重试，保留已填写的设定

## 二、写作台保存机制（P0）

### 问题

- 自动保存 2 秒延迟，用户可能关闭页面前未保存
- 无保存状态提示（用户不知道是否已保存）
- 无断网/冲突处理

### 任务

- [ ] **2.1 保存状态可视化**
  - 编辑器底部显示："已保存 / 保存中 / 未保存"
  - 使用 `navigator.sendBeacon` 在页面关闭前强制保存

- [ ] **2.2 防抖优化**
  - 输入停止 1 秒后触发保存（当前 2 秒）
  - 切换集数时立即保存当前集

- [ ] **2.3 离线保护**
  - 监听 `online/offline` 事件
  - 离线时写入 `localStorage` 缓存，恢复后同步

- [ ] **2.4 版本标记**
  - 每次保存更新 `updatedAt`
  - 显示"最后保存于 X 分钟前"

## 三、角色/场地拖拽插入（P0）

### 问题

- `ProjectWrite.vue` 有 dragstart/drop 事件，但格式不规范
- 拖拽后光标位置可能错乱
- 无视觉反馈（用户不知道拖成功了）

### 任务

- [ ] **3.1 规范插入格式**
  - 角色：`\n@${角色名}\n`
  - 场地：`\n#${场地名} | 日内\n`
  - 钩子：直接插入到光标位置

- [ ] **3.2 拖拽视觉反馈**
  - 拖拽时显示 ghost 元素
  - 编辑器区域显示 drop 区域高亮
  - 插入后闪烁高亮插入的文本 1 秒

- [ ] **3.3 快速插入按钮**
  - 素材面板每个角色/场地旁添加 "+" 按钮
  - 点击直接插入到当前光标位置（比拖拽更快）

## 四、AI 助手短剧专用 Prompt（P1）

### 问题

- 当前 AI 指令调用通用 chat API，无短剧专用上下文
- 续写可能偏离角色人设
- 生成的钩子可能不符合短剧节奏

### 任务

- [ ] **4.1 续写 Prompt**
  - 注入项目角色卡（名称、性格、关系）
  - 注入当前集钩子/悬念约束
  - 要求：短剧语感、一句一顶撞、情绪词强化

- [ ] **4.2 润色 Prompt**
  - 将叙述性文字改为对白
  - 用"怒斥"代替"说"，用"冷笑"代替"回答"
  - 压缩冗余，保留核心冲突

- [ ] **4.3 钩子生成 Prompt**
  - 分析当前集已写内容
  - 生成 3 个不同类型的结尾钩子
  - 标注每个钩子的"付费转化潜力"

- [ ] **4.4 冲突强化 Prompt**
  - 识别平淡对白段落
  - 改为对抗式对白（A质疑→B反驳→A威胁→B反击）
  - 保持角色人设不崩

- [ ] **4.5 投流文案 Prompt**
  - 从剧本中提取"高光 10 秒"
  - 生成 5 条不同风格的广告文案
  - 标注每条文案的"情绪钩子"

## 五、集数导航状态可视化（P1）

### 问题

- 导航只显示集数，无写作进度
- 不知道哪些集已完成、哪些为空

### 任务

- [ ] **5.1 进度计算**
  - `writeStatus = completed`：有 content 且字数 > 500
  - `writeStatus = writing`：有 content 但字数 < 500
  - `writeStatus = pending`：无 content

- [ ] **5.2 导航视觉状态**
  - 已完成：绿色圆点
  - 写作中：黄色圆点
  - 未开始：灰色圆点
  - 付费卡点：金色边框

- [ ] **5.3 整体进度条**
  - 导航上方显示："已完成 X/40 集"
  - 项目卡片上显示进度条

## 六、钩子检测器（P2）

### 问题

- 编剧可能忘记在结尾埋钩子
- 付费卡点的集数可能钩子强度不够

### 任务

- [ ] **6.1 规则检测**
  - 检查最后 3 行是否包含：疑问句、身份词、威胁词、意外词
  - 关键词库："是你？"、"不可能"、"为什么"、"竟然"、"秘密"、"真相"

- [ ] **6.2 视觉警告**
  - 无钩子时：编辑器底部显示红色警告 "⚠️ 本集结尾缺少钩子"
  - 提供 "一键插入钩子" 按钮（从钩子库随机选）

- [ ] **6.3 AI 检测（后续）**
  - 调用 LLM 评估钩子强度 1-10
  - 低于 6 分建议优化
