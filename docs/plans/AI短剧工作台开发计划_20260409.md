# AI短剧工作台·开发计划

## 项目概览

| 项目         | 内容                                                               |
| ------------ | ------------------------------------------------------------------ |
| **项目名称** | AI短剧工作台（天工开物）                                           |
| **项目结构** | Monorepo（前后端同仓库，TypeScript类型共享）                       |
| **MVP目标**  | 跑通剧本→分镜→视频生成→视频合成全流程                              |
| **技术栈**   | Vue3 + Fastify + PostgreSQL + Redis + BullMQ                       |

## 阶段一：项目脚手架搭建

### 1.1 Monorepo 结构设计

```
dreamer/
├── packages/
│   ├── frontend/          # Vue 3 前端应用
│   ├── backend/            # Fastify 后端服务
│   └── shared/             # 共享类型定义
├── docker/                 # Docker Compose 配置
│   └── docker-compose.yml  # PostgreSQL + Redis + MinIO
├── scripts/                # 辅助脚本
├── package.json            # 根 workspace 配置
└── README.md
```

### 1.2 前端脚手架

- **框架**: Vue 3 + TypeScript + Vite
- **UI库**: Naive UI（比Element Plus更现代）
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: ky 或 ofetch

### 1.3 后端脚手架

- **框架**: Fastify + TypeScript
- **ORM**: Prisma
- **任务队列**: BullMQ + ioredis
- **对象存储**: MinIO SDK
- **AI SDK**: DeepSeek SDK（火山引擎用原生fetch）

### 1.4 共享包设计

- `packages/shared/src/types/` - 共享TypeScript类型
- `packages/shared/src/schemas/` - Zod验证schema（前后端共用）

## 阶段二：数据库设计与建模

### 2.1 核心数据模型

```prisma
model Project {
  id          String   @id @default(cuid())
  name        String
  description String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  episodes    Episode[]
  characters  Character[]
}

model Episode {
  id          String   @id @default(cuid())
  projectId   String
  project     Project  @relation(fields: [projectId], references: [id])
  episodeNum  Int
  title       String?
  script      Json?    // 存储结构化剧本JSON
  scenes      Scene[]
  createdAt   DateTime @default(now())
}

model Character {
  id          String   @id @default(cuid())
  projectId   String
  project     Project  @relation(fields: [projectId], references: [id])
  name        String
  description String?
  avatarUrl   String?  // 定妆照URL
  versions    Json?    // 存储不同服装版本的参考图
}

model Scene {
  id          String   @id @default(cuid())
  episodeId   String
  episode     Episode  @relation(fields: [episodeId], references: [id])
  sceneNum    Int
  description String?  // 场景描述
  prompt      String   // 生成视频的最终提示词
  status      String   @default("pending") // pending/processing/completed/failed
  createdAt   DateTime @default(now())
  tasks       VideoTask[]

  // 最终成品（从selected VideoTask汇总）
  // videoUrl从选中的VideoTask获取
}

model VideoTask {
  id          String   @id @default(cuid())
  sceneId     String
  scene       Scene    @relation(fields: [sceneId], references: [id])
  model       String
  status      String
  prompt      String
  cost        Float?
  duration    Int?
  videoUrl    String?
  thumbnailUrl String?
  errorMsg    String?
  isSelected  Boolean  @default(false)  // 是否被选为最终版本
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

// 视频合成
model Composition {
  id          String   @id @default(cuid())
  projectId   String
  title       String
  duration    Int      @default(0)
  width       Int      @default(1080)
  height      Int      @default(1920)
  status      String   @default("draft")
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  segments    Segment[]
  voiceover   String?  // 配音文件URL
  bgm         String?  // 背景音乐URL
  subtitles   String?  // 字幕文件URL
  outputUrl   String?  // 最终成品URL
}

model Segment {
  id            String      @id @default(cuid())
  compositionId String
  composition  Composition @relation(fields: [compositionId], references: [id])
  sceneId       String      // 关联的Scene
  order         Int         // 时间轴顺序
  startTime     Float       // 起始时间（秒）
  endTime       Float       // 结束时间（秒）
  transition    String?     // 转场效果
}
```

### 2.2 Worker处理流程

```
接收Job → 更新Scene状态为processing
  ↓
根据model调用对应视频生成API（Wan 2.6 / Seedance 2.0）
  ↓
轮询API直到视频生成完成（或失败）
  ↓
下载视频文件，上传至MinIO/OSS
  ↓
更新数据库：Scene状态改为completed，记录videoUrl、cost、duration
  ↓
（可选）通过WebSocket/SSE推送完成通知
```

启动以下服务：

- **PostgreSQL 16** - 主数据库
- **Redis 7** - 缓存+队列
- **MinIO** - 对象存储（兼容S3 API）

## 阶段五：AI工作流集成

> **核心原则：AI生成 → 人工干预/修改 → 确认执行。每一步均可人工干预，无黑盒自动化。**

### 5.0 人工干预节点总览

| 环节 | AI操作 | 人工干预 |
|------|--------|---------|
| 剧本 | AI扩写生成 | 任意编辑、调整场景/对话/动作 |
| 角色 | AI生成定妆照 | 上传/替换参考图、修改描述 |
| 分镜 | AI辅助拆解、提示词优化 | 增删改、手动调整提示词 |
| 视频 | AI生成视频片段 | 重生成、不满意可反复提交 |
| 合成 | FFmpeg导出 | 时间轴调整、配乐/字幕编辑 |

### 5.1 DeepSeek API（剧本生成）

- **端点**：`https://api.deepseek.com/v1/chat/completions`
- **认证**：Bearer Token
- **SDK**：与OpenAI SDK兼容，直接使用 `openai` npm包，修改 `baseURL`

**成本优化**：
- 固定的角色描述、风格设定放在 **System Prompt** 中，利用上下文缓存降低90%输入成本
- 非紧急任务安排到**闲时调用**（DeepSeek闲时价格五折，通常23:00-07:00）

### 5.2 Wan 2.6 API（低成本试错）

- **供应商**：Atlas Cloud
- **调用方式**：REST API，提交提示词 → 返回任务ID → 轮询获取结果
- **成本**：约 **$0.07/秒**，生成10秒视频仅$0.7
- **用途**：大规模快速验证分镜构图，适合试错迭代

### 5.3 Seedance 2.0 API（高光精修）

- **供应商**：火山引擎（需企业认证）
- **调用方式**：REST API，支持文本生视频、图生视频
- **成本**：约 **1元/秒**（46元/百万tokens，约2.058 tokens/秒）
- **并发限制**：单账号最大并发10个任务
- **用途**：最终高光片段、成品级视频

### 5.4 三阶段分镜生成工作流

对于每个分镜，采用**试错→筛选→定稿**的严谨流程：

```
【试错阶段】用 Wan 2.6 生成 3-5 个低成本草稿（$0.7/10秒）
  ↓
【筛选阶段】人工挑选最满意的草稿构图
  ↓
【定稿阶段】基于选中草稿的构图，用 Seedance 2.0 生成最终高质量片段（约 $10/10秒）
  ↓
【成品】获得高质量分镜片段，可进入合成时间轴
```

**成本估算**：单个分镜（10秒）= $0.7×3~5（试错） + $10（定稿）≈ $12~15

**VideoTask表支持多版本**：
- 每个Scene可创建多个VideoTask
- 标记 `isSelected` 字段表示被选中的草稿/定稿
- 未选中的草稿可保留用于对比

## 注意事项

### 8.1 API密钥安全

- **严禁在前端代码或环境变量中暴露任何API密钥**
- 所有第三方API调用必须经过后端代理
- 后端使用 `.env` 文件管理密钥，生产环境使用云厂商密钥管理服务

### 8.2 成本控制

- 数据库记录每次生成任务的 `cost` 字段，便于核算
- 前端在提交高成本任务时弹窗确认
- 可设置单日/单用户成本上限，防止意外超支

### 8.3 视频生成异步处理

- 视频生成任务耗时较长（10秒视频可能需要数分钟），必须走任务队列
- 前端采用**轮询**（每2-3秒查询一次状态）作为基础方案，稳定可靠
- 后续可升级为 SSE/WebSocket 实时推送

### 8.4 角色一致性

- 尽量使用 **图生视频**（传入角色定妆照作为参考图），而非纯文本描述
- 为同一角色建立多个服装版本的参考图，在分镜中按场景选择

### 8.5 内容审核

- 若平台对外开放，需建立内容审核机制，过滤违规提示词和生成的视频内容
- AI生成内容需按广电要求进行标识
