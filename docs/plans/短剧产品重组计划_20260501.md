# Dreamer 短剧产品重组计划

> 日期：2026-05-01
> 目标：基于短剧编剧/MCN/投流团队的核心需求，重新梳理产品架构，砍掉冗余功能，聚焦"情绪过山车设计器"定位

---

## 一、现状诊断：为什么"混乱"

### 1.1 功能层面：贪多求全，主线不清

| 问题                  | 表现                                                                                                         |
| --------------------- | ------------------------------------------------------------------------------------------------------------ |
| **双轨剧本系统**      | `Script`（独立剧本表）和 `Episode.script`（项目内分集剧本）并存，数据不互通，用户困惑                        |
| **三个编辑器**        | Studio.vue（AI写作工作室）、EditorPage.vue（Tiptap剧本编辑器）、ProjectScript.vue（项目内剧本）功能重叠      |
| **视频生成喧宾夺主**  | 大量代码投入在 Seedance/Wan2.6 视频生成、Composition 合成、Take 管理，但短剧的核心是**剧本**，视频是下游执行 |
| **Pipeline 过度工程** | PipelineJob + PipelineStepResult 设计了完整流水线，但用户真正需要的是"快速起量"，不是全自动                  |
| **记忆系统悬空**      | MemoryItem/MemorySnapshot 建好了表，但前端几乎没有调用入口，编剧感受不到价值                                 |

### 1.2 架构层面：模型正确，产品错位

- **技术架构优秀**：Provider-Registry 模式、BullMQ 任务队列、成本追踪、多模型切换——这些都是好底子
- **产品架构失焦**：用户打开平台，不知道该先做什么。是写剧本？导入剧本？生成视频？还是跑 Pipeline？

### 1.3 用户路径断裂

当前用户路径：

```
Dashboard → 新建项目 → 项目详情（N个tab）→ 剧本编辑 → 分集 → 分镜 → 视频生成 → 合成
```

问题：

1. 编剧来平台的第一诉求是"快速出一部能投流的短剧大纲+前3集"
2. 当前路径要求用户先理解"项目-分集-分镜-镜头"四级结构，门槛太高
3. 没有"模板→大纲→剧本"的快速通道
4. 没有钩子/卡点/爽点等短剧核心概念的显性表达

## 二、重组原则

### 2.1 核心原则："剧本优先，投流导向"

```
┌─────────────────────────────────────────────────────────────┐
│                      用户价值金字塔                           │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: 投流赋能（差异化竞争力）                            │
│  - 高光片段自动提取 → 投流文案生成                            │
│  - AB测试剧本片段 → 数据反哺创作                              │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: 剧本工业化（核心壁垒）                              │
│  - 爆款模板库 → 40集大纲一键生成                              │
│  - 钩子/卡点检测器 → 每集结尾强制埋钩                         │
│  - 情绪曲线追踪 → 平缓即警告                                  │
│  - 冲突密度仪表盘 → 一句一顶撞                               │
├─────────────────────────────────────────────────────────────┤
│  Layer 1: 基础创作（必须做好）                                │
│  - 项目/剧本管理                                              │
│  - AI 辅助写作（续写/润色/扩写/缩写）                         │
│  - 角色/场地管理                                              │
│  - 导出竖屏脚本格式                                           │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 砍掉什么（做减法）

| 功能                                  | 决策                            | 理由                                                 |
| ------------------------------------- | ------------------------------- | ---------------------------------------------------- |
| `Script` 独立表 + Studio/EditorPage   | **合并到 Project-Episode 体系** | 双轨制是混乱根源，剧本必须依附项目才有上下文         |
| 视频生成（Take/Composition/Seedance） | **降级为扩展模块，MVP不展示**   | 短剧编剧的核心工具是"写"，不是"拍"；视频交给专业团队 |
| Pipeline 全自动流水线                 | **简化为一键生成大纲+前3集**    | 编剧需要控制感，全自动反而失去信任                   |
| 分镜/镜头级管理（Shot/Storyboard）    | **MVP隐藏，后续迭代**           | 先让剧本跑通，再往下游延伸                           |
| TTS 语音合成                          | **保留但降级**                  | 有用但不是核心，先保证剧本质量                       |

### 2.3 保留什么（做精做透）

| 功能          | 强化方向                                                        |
| ------------- | --------------------------------------------------------------- |
| 项目-分集体系 | 增加"模板选择→大纲生成→逐集写作"主线                            |
| AI 写作助手   | 从通用续写升级为"短剧专用Agent"（钩子生成、卡点设计、冲突强化） |
| 角色/场地库   | 与剧本写作深度联动（写作时自动引用角色卡）                      |
| 成本追踪      | 保留，增加"每集成本"维度                                        |
| 任务中心      | 保留，简化展示                                                  |

    ├── 顶部：项目信息 + 题材标签 + 视觉风格
    ├── 左侧：集数导航（1-40集，显示每集钩子状态）
    ├── 中间：剧本编辑器（核心区域）
    │   ├── 编辑器本体
    │   ├── 浮动AI助手（续写/润色/生成钩子/检测卡点）

    ├── 右侧：素材面板

    └── 底部：全局操作
        ├── 导出竖屏脚本
        ├── 生成投流文案（新）

模板库（Templates）—— 新增一级导航
├── 爆款结构模板（赘婿逆袭/霸总甜宠/重生复仇/悬疑惊悚）

├── 项目统计
├── AI成本明细
└── 投流潜力评估（新）

```prisma
// 爆款模板
model DramaTemplate {
  id          String   @id @default(cuid())
  name        String   // "赘婿逆袭流"
  category    String   // "男频/逆袭"
  description String   // 模板说明
  structure   Json     // 节拍结构：[{ act: 1, beat: "隐藏身份", duration: "3集" }, ...]
  exampleOutline String? @db.Text // 示例大纲
  isBuiltin   Boolean  @default(true)
  createdAt   DateTime @default(now())
}

// 项目大纲（独立于Episode，用于模板生成阶段）
model ProjectOutline {
  id          String   @id @default(cuid())
  projectId   String   @unique
  project     Project  @relation(fields: [projectId], references: [id], onDelete: Cascade)
  templateId  String?
  episodes    Json     // [{ episodeNum, title, synopsis, hook, cliffhanger, paywall? }]
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

// 钩子库（用户收藏+系统预设）
model HookTemplate {
  id          String   @id @default(cuid())
  userId      String?  // null = 系统预设
  category    String   // "悬念/威胁/反转/情感"
  content     String   // 钩子文本模板
  tags        String[]
  usageCount  Int      @default(0)
  createdAt   DateTime @default(now())
}

// 分集质量评分（AI评估）
model EpisodeScore {
  id              String   @id @default(cuid())
  episodeId       String   @unique
  episode         Episode  @relation(fields: [episodeId], references: [id], onDelete: Cascade)
  hookStrength    Int      // 钩子强度 1-10
  conflictDensity Int      // 冲突密度 1-10
  emotionVariance Int      // 情绪震荡 1-10
  pacingScore     Int      // 节奏得分 1-10
  paywallPotential Int     // 付费潜力 1-10
  suggestions     String?  @db.Text
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}
```

## 四、核心功能重新定义

### 4.1 模板库 → 40集大纲生成器

**用户路径**：

1. 选择模板（如"赘婿逆袭流"）
2. 填写核心设定（主角名、身份、核心矛盾）
3. AI 生成 40 集大纲（每集：标题 + 一句话梗概 + 钩子 + 是否付费点）
4. 可视化大纲编辑器（时间线形式，可拖拽调整）
5. 点击"开始写作"→ 创建项目，进入写作模式

**技术实现**：

- 后端：`DramaTemplate` 表 + `generateOutline` service
- 前端：模板选择页 + 大纲编辑器（类似甘特图/时间线）
- AI Prompt：专项训练，输出严格格式 JSON

### 4.2 剧本编辑器 → 短剧专用写作台

**核心区域**：

```
┌──────────────────────────────────────────────────────────────┐
│  集数导航  [1] [2] [3] [4] ... [40]     [导出] [AI检测]       │
├──────────────────────────────────────────────────────────────┤
│  本集信息：标题 [________]  钩子 [________]  结尾悬念 [____]   │
├──────────────┬──────────────────────────────┬────────────────┤
│              │                              │                │
│  角色面板    │      剧本编辑器              │   AI助手面板   │
│  [拖拽插入]  │      （Fountain简化格式）    │   [续写]       │
│              │                              │   [润色]       │
│  场地面板    │      情绪曲线图（底部）      │   [生成钩子]   │
│  [拖拽插入]  │      冲突密度条（底部）      │   [检测卡点]   │
│              │                              │   [投流文案]   │
│  钩子库      │                              │                │
│  [一键插入]  │                              │                │
│              │                              │                │
├──────────────┴──────────────────────────────┴────────────────┤
│  状态栏：字数 / 预计时长 / 冲突密度: ████░░ / 情绪平缓警告 ⚠️  │
└──────────────────────────────────────────────────────────────┘
```

**编辑器格式**（Fountain 简化）：

```
# 第3场  总裁办公室  日

@林婉儿（愤怒）
你凭什么开除我？

@顾北辰（冷笑）
凭我是这家公司的主人。

[钩子提示：此处可加强冲突，建议加入"身份揭露"元素]
```

### 4.3 AI 助手 → 短剧专用 Agent

**指令体系**：
| 指令 | 功能 |
|------|------|
| `续写` | 基于上下文+角色卡，续写后续对白 |
| `润色` | 强化情绪词，压缩叙述，增加网感 |
| `生成钩子` | 为当前集生成3个结尾钩子方案 |
| `检测卡点` | 分析当前集是否适合作为付费断点 |
| `冲突强化` | 将平淡对白改为对抗式对白 |
| `情绪曲线` | 分析当前集情绪波动，标出平缓区 |
| `投流文案` | 从本集提取高光，生成20条广告文案 |
| `查重检测` | 与热榜短剧对比，标出雷同段落 |

### 4.4 导出 → 竖屏拍摄脚本

**导出格式**（Markdown，适配手机阅读）：

```markdown
# 《剧名》第X集 拍摄脚本

---

## 场景1：总裁办公室 | 日内

**镜头1** | 特写 | 2秒
林婉儿眼眶泛红，手指攥紧辞职信。

**镜头2** | 中景 | 3秒  
顾北辰靠在皮椅上，嘴角微扬。
顾北辰："你走了，谁来给我煮咖啡？"

**镜头3** | 近景 | 2秒
林婉儿（内心OS）：他到底什么意思...

---

**本集时长**：约45秒
**情绪标签**：委屈→愤怒→困惑
**投流亮点**：第2句台词可剪为"霸总语录"
```

## 七、AI Prompt 体系重组

### 7.1 大纲生成 Prompt

```
你是一位资深短剧编剧，擅长"赘婿逆袭"题材。
请根据以下核心设定，生成40集短剧大纲：

核心设定：
- 主角：{name}，{identity}
- 核心矛盾：{conflict}
- 目标受众：{audience}

要求：
1. 每集必须包含：标题、一句话梗概、开头钩子、结尾悬念
2. 第10、20、30集设置为付费卡点（剧情断在最关键处）
3. 遵循"隐藏身份→被羞辱→身份揭露→打脸→新危机"循环
4. 每3集必须有一个爽点（打脸/反转/身份揭露）
5. 输出JSON格式
```

### 7.2 剧本写作 Prompt

```
你是一位短剧对白专家。请根据以下信息续写剧本：

上下文：
- 项目：{projectName}，{genre}题材
- 当前集：第{episodeNum}集，{episodeTitle}
- 本集钩子：{hook}
- 目标结尾悬念：{cliffhanger}

角色卡：
{characters}

已写内容：
{existingContent}

续写要求：
1. 短剧语感：一句一顶撞、一段一转折
2. 情绪词强化：用"怒斥"代替"说"，用"冷笑"代替"回答"
3. 每段对白不超过3句
4. 保持角色人设一致
5. 向结尾悬念推进
```

### 7.3 钩子生成 Prompt

```
请为以下短剧集生成3个结尾钩子方案：

已写内容：
{content}

要求：
1. 每个钩子必须在最后3行内制造强烈悬念
2. 类型可选：身份揭露/意外闯入/新威胁/情感冲击
3. 钩子要让人忍不住想看下一集
4. 标注每个钩子的"付费转化潜力"（高/中/低）
```
