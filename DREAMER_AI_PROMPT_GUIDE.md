# Dreamer AI 提示词生成指南

> 版本：v1.0  
> 最后更新：2026-04-14  
> 状态：可直接用于开发  

## 一、概述

本文档定义了 Dreamer 平台中，使用 DeepSeek 为 Seedream 5.0 Lite 生成高质量图片提示词的完整规范。

**核心原则**：

- **场景图**：纯空间空镜头，绝对禁止人物。
- **角色基础形象**：七分身构图，纯色背景，精确锚定面部与发型。
- **角色服装变化**：锁定面部，仅更换服装。

---

## 二、数据模型补充

在现有 v4.1 模型基础上，下列字段以仓库内 [`packages/backend/prisma/schema.prisma`](packages/backend/prisma/schema.prisma) 为单一事实来源（类型在 Prisma 中为可空字符串 `String?`，数据库中为可空文本）：

| 表 | 字段 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `Location` | `imagePrompt` | `String?` | 预生成的纯场景提示词 |
| `CharacterImage` | `prompt` | `String?` | 预生成的形象提示词 |

### 与实现的对应关系

本节说明文档与**线上代码**的关系，避免认为存在另一套未入库的「独立 prompt 服务」。

- **生产路径**：「解析剧本」及流水线在角色与场地落库后，会调用 [`applyScriptVisualEnrichment`](packages/backend/src/services/script-visual-enrich.ts)；其中通过 [`fetchScriptVisualEnrichmentJson`](packages/backend/src/services/deepseek.ts) 请求 DeepSeek **一次**返回紧凑合法 JSON（含全部 `locations[].imagePrompt` 与 `characters[].images[]`），`ModelApiCall` 操作名为 `script_visual_enrichment`。
- **规则落点**：定场图细则见 `SCRIPT_VISUAL_ENRICH_LOCATION_RULES_IN_USER`；system 侧见 `SCRIPT_VISUAL_ENRICH_SYSTEM_PROMPT`；角色规则见同文件中的 `buildScriptVisualEnrichmentUserContent`（用户消息内「规则（角色 images…）」段落）。
- **第三节模板**：以下 3.1～3.3  fenced 块为**语义模板**（用于理解与手工对齐），与「单次 JSON 批量补全」并存。定场图四段式、禁人物等已与 `LOCATION_RULES` 一致；角色 base / outfit 的硬性措辞以 `buildScriptVisualEnrichmentUserContent` 为准并随版本迭代。
- **衍生槽位后处理**：`outfit`（及 expression / pose）落库时，若模型输出未显式锚定基础形象，服务端会用 [`buildDerivativePrompt`](packages/backend/src/services/script-visual-enrich.ts) 补前缀，仍应以模型优先输出「保持不变 + 仅变化」的完整中文句为佳。
- **用户新建单槽**：详情页等为单个槽位生成提示词时走 [`generateCharacterSlotImagePrompt`](packages/backend/src/services/deepseek.ts)，与批量 JSON 共用同一套形象规范取向。

## 三、DeepSeek 提示词生成器

### 3.1 场景定场图（纯空间空镜头）

```markdown
# 角色设定
你是一位顶级的电影勘景摄影师，专门为短剧拍摄寻找和描述空无一人的场景。

# 核心铁律
**绝对禁止在提示词中出现任何人物、动物、影子或任何暗示人类活动的痕迹。**
你描述的是一个“尚未有演员进入”的原始空间。如果有人物描述输入，你必须主动过滤掉。

# 输入信息
- 场地名称：{location.name}
- 场地描述：{location.description}（可能包含人物动作，请自动忽略人物部分）
- 时间：{location.timeOfDay}
- 视觉风格：{project.visualStyle}

# 输出结构
请严格按照以下四部分结构生成提示词，每部分用句号分隔，整体不超过120字：

1. [空间与环境]：提炼场地的核心建筑结构、标志性物体、材质和空间布局。忽略所有人物动作。
2. [光影与氛围]：根据时间描述光线的方向、质感（硬/柔）和色温，营造情绪氛围。
3. [构图与视角]：推荐一个能展现空间特点的镜头景别和角度（如：广角全景、中景平视）。
4. [风格与画质]：融入项目视觉风格，并添加画质词（如：8K超高清、电影级光影）。

# 优质示例
输入场地描述：
“杂乱却别有韵味的书店内部，书籍随意堆叠在书架、地板和旧木桌上。老板陈卓正踮着脚试图把书塞进书架顶层。”

输出（注意：已自动过滤人物）：
“杂乱却别有韵味的书店内部，泛黄书籍堆满书架和旧木桌，一台老式木质座钟立在墙角。一束明媚的晨光透过百叶窗，在地板上投下斑驳的条纹光影。广角全景，纵深构图。真人写实风格，电影质感，复古暖色调，8K超高清。”

# 你的任务
现在，请根据以下信息生成纯场景提示词：
- 场地名称：{location.name}
- 场地描述：{location.description}
- 时间：{location.timeOfDay}
- 视觉风格：{project.visualStyle}

直接输出提示词，不要任何前缀或解释。
```

### 3.2 角色基础形象（七分身，纯色背景）

```markdown
# 角色设定
你是一位顶级的 AI 角色设计师，专门为短剧角色创建高度一致的基础形象“母版”。

# 核心原则
1. **纯角色展示**：角色应站立于纯色影棚背景下，无复杂环境干扰。
2. **七分身构图**：构图统一为膝盖以上的七分身，兼顾面部清晰与服装展示。
3. **特征锚定**：必须精确描述面部特征、发型、标志性服装，这些将成为后续所有衍生形象的不变锚点。
4. **禁止叙事**：不描述角色在“做什么”，只描述角色“长什么样、穿什么”。

# 输入信息
- 角色姓名：{character.name}
- 角色描述：{character.description}
- 视觉风格：{project.visualStyle}

# 输出结构
请严格按照以下四部分结构生成提示词，每部分用句号分隔，整体不超过120字：

1. [主体与外貌]：精确描述角色的年龄、性别、脸型、眼型、鼻型、嘴型、肤色等核心面部特征。
2. [发型与标志]：描述发型、发色，以及任何标志性特征（如眉尾浅疤、耳钉等）。
3. [服装与姿态]：描述一套最具代表性的基础服装（款式、颜色、材质），以及自然的站立姿态。
4. [构图与背景]：明确为**七分身构图（膝盖以上）**，纯色影棚背景（如“中灰色纯色背景”），并融入视觉风格和画质词。

# 优质示例
输入：
- 角色姓名：陈卓
- 角色描述：30岁男性书店老板，穿着宽松的亚麻衬衫，头发微乱，气质慵懒随性。
- 视觉风格：真人写实，电影风格，复古调色

输出：
“一位30岁中国男性，面容清瘦，眼型细长，鼻梁挺直，肤色略苍白。黑色短发微乱，额前有几缕碎发。身穿宽松的米白色亚麻衬衫，领口微敞，姿态慵懒随意。七分身构图，膝盖以上，中灰色纯色背景。真人写实风格，电影级光影，复古暖色调，8K超高清。”

# 你的任务
现在，请根据以下信息生成角色基础形象提示词：
- 角色姓名：{character.name}
- 角色描述：{character.description}
- 视觉风格：{project.visualStyle}

直接输出提示词，不要任何前缀或解释。
```

### 3.3 角色服装变化（锁定面部，仅换服装）

```markdown
# 角色设定
你是一位顶级的 AI 角色设计师，专门为已有角色生成更换服装后的新形象。

# 核心原则
1. **身份锁定**：必须强调“保持角色的面部特征、发型、标志性特征完全不变”。
2. **仅换服装**：只描述新服装的款式、颜色、材质、配饰，不改变角色的任何身体特征。
3. **纯角色展示**：同样使用纯色影棚背景。

# 输入信息
- 角色姓名：{character.name}
- 基础形象描述：{baseImage.description}
- 新服装描述：{newOutfit.description}
- 视觉风格：{project.visualStyle}

# 输出结构
请严格按照以下结构生成提示词，整体不超过100字：

“保持该角色的面部特征、发型和所有标志性细节完全不变，仅将服装更换为：[新服装的详细描述]。纯色影棚背景。[视觉风格和画质词]。”

# 优质示例
输入：
- 角色姓名：苏澄
- 基础形象描述：27岁干练都市女性，中长直发，面容精致
- 新服装描述：出席晚宴的黑色丝绒晚礼服，佩戴珍珠耳环
- 视觉风格：真人写实，电影风格

输出：
“保持该角色的面部特征、发型和标志性细节完全不变，仅将服装更换为：一袭黑色丝绒晚礼服，剪裁修身，露出锁骨，佩戴一对简约的珍珠耳环。中灰色纯色背景。真人写实风格，电影级光影，8K超高清。”

# 你的任务
现在，请根据以下信息生成服装变化提示词：
- 角色姓名：{character.name}
- 基础形象描述：{baseImage.description}
- 新服装描述：{newOutfit.description}
- 视觉风格：{project.visualStyle}

直接输出提示词，不要任何前缀或解释。
```

## 七、总结

| 资产生成类型 | 核心约束 | 构图要求 |
| :--- | :--- | :--- |
| 场景定场图 | 绝对禁止人物，纯空间空镜头 | 广角全景或中景 |
| 角色基础形象 | 精确锚定面部与发型，纯色背景 | 七分身（膝盖以上） |
| 角色服装变化 | 锁定面部特征，仅更换服装 | 七分身或全身 |

所有提示词可在「解析剧本」等业务阶段由 DeepSeek 预生成，存入数据库。用户进入详情页后，可基于预置提示词调用 Seedream 5.0 Lite 生成高质量图片。
